Drawing a background

In this game the player is going to be a rocket flying through space. Their job will be to stay alive as long as possible so they can score points, but before we can do that we need to set the scene by creating a background that shows outer space.

I’ve provided a background graphic for you called space@2x.jpg, so we need to add that to our Xcode project so that iOS knows what to show. It’s inside the Assets > Project1 folder that came with this book.

Tip: The @2x part of image filenames is a little bit of magic. Different iOS devices have different resolutions, but mostly developers don’t want to think about that when writing code. So, we’ll ask Xcode to load space.jpg and it will automatically look for that image at the correct resolution for the current device. iPads use retina resolution, which is 2x regular resolution, so when we ask for space.jpg iOS will actually load space@2x.jpg.

On the left of the Xcode window is the project navigator, where you add files. If you don’t see it already, press Cmd+1 to activate it. One of those files is called Assets.xcassets, which is an asset catalog – a place where we store all our images. Please select that now, and you’ll see it already has one empty asset called “AppIcon”.

I’d like you to add space@2x.jpg to that asset catalog – select it in Finder, then drag it below “AppIcon”.

Now that space@2x.jpg is part of our project we can start using it in our game. In SpriteKit, each part of your game is called a scene, and we have one ready to use already called GameScene.swift. Please select that using the project navigator, and you’ll see it opened for editing.

There are four methods in there already, which are pieces of functionality that get called by SpriteKit when something interesting happens. They are:

  • The didMove(to:) method, which is called when your game scene is ready to run.
  • The touchesBegan() method, which is called when the user touches the screen.
  • The touchesEnded() method, which is called when the user stops touching the screen.
  • The update() method, which is called before each frame is rendered.

Each of those methods has a comment inside, which is the green text that starts with //. These comments are ignored by Swift, but we use them to help explain to other programmers what’s going on.

We want to set up our new background when the game scene is ready to run, so we need to put it into the didMove(to:) method. Add this code just below the comment inside that method:

let background = SKSpriteNode(imageNamed: "space.jpg")
background.zPosition = -1
addChild(background)

If you’ve put it in the right place, here’s how your code should look:

override func didMove(to view: SKView) {
    // this method is called when your game scene is ready to run
    let background = SKSpriteNode(imageNamed: "space.jpg")
    background.zPosition = -1
    addChild(background)
}

Those three lines of code do three distinct things.

The first line creates a new SKSpriteNode, which is SpriteKit’s way of drawing images on the screen. SpriteKit thinks about your scene in terms of nodes, which are a collection of objects that form a neat hierarchy. In this case we want a sprite node, which is a node that stores pictures, so we’re going to make it load from the space@2x.jpg picture we added earlier.

Just creating a new sprite node isn’t enough, we need to show it too. That’s what the third line does: it adds the new sprite node as a child of our game scene, which causes it to be displayed.

As for the second line, that’s a bit special: Z position lets us decide whether a sprite should be drawn on top of or below other sprites. Using higher numbered Z positions makes sprites draw on top of others, so using -1 means “draw behind others.” These numbers are all relative – we could have used -1000 if we wanted.

That’s enough code for now, so press Cmd+R to ask Xcode to build and run your code in the simulator. Hopefully you have an iPad connected so Xcode can deploy the app directly there. If not, there is a virtual iPad called the simulator, but it’s extremely slow even when you’re using high-performance Macs – it’s strongly recommended that you use a real iPad.

If everything has worked well you should see the iPad’s screen filled with a picture of outer space – a good start!
