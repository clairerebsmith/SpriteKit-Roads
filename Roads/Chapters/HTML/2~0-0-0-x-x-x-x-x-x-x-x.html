Making a grid of animals

The next step takes a little more thought, but it uses the same SKSpriteNode objects you just learned about: we’re going to fill the screen with animals, lined up neatly in a 12x8 grid.

Look in the assets for this project and find elephant@2x.png, giraffe@2x.png, hippo@2x.png, monkey@2x.png, panda@2x.png, parrot@2x.png, penguin@2x.png, pig@2x.png, rabbit@2x.png, and snake@2x.png, then copy them into your project’s asset catalog so we can use them. Later on we’ll be showing all of those images, but for now we’re just going to show one so we can see the grid looks correct – elephant@2x.png.

Each of the animals is 64x64 points in size, but it doesn’t look nice having them directly side by side so we’re going to make the grid be 80x80 in side. It also doesn’t look great having them go right to the edge of the screen, so we’re going to add some padding to the edges.

Our SpriteKit scene is 1024x768, but as I said earlier SpriteKit measures its node positions from their center. So, the left edge of our screen is -512. Rather than go right to the edge, we’re going to start our grid at X:-440 so there’s some space on the left, and vertically we’ll use Y:-320 so there’s some space at the bottom too.

From those X and Y offsets, we can create our grid using a simple loop: each of the animals will have the X position of X offset + (column number * 80) and the Y position of Y offset + (row number * 80).

Add this method below didMove(to:) now:

func createGrid() {
    let xOffset = -440
    let yOffset = -320

    for row in 0 ..< 8 {
        for col in 0 ..< 12 {
            let item = SKSpriteNode(imageNamed: "elephant")
            item.position = CGPoint(x: xOffset + (col * 80), y: yOffset + (row * 80))
            addChild(item)
        }
    }
}

Now we just need to call that method by adding one line to the end of the didMove(to:) method:

createGrid()

Try running the app now and you’ll see a grid full of animals. Of course, we don’t want them all to be showing because that would be rather hard, so next we’re going to be looking at difficulty levels…
