Creating the player

The next step is to create the player’s submarine on the screen. I’ve provided another picture for this in the assets that came with this book, and just like with the background our first step is to add the new picture to our project’s asset catalog.

So, find the file player-submarine@2x.png in the assets directory for this project, then drag it into your asset catalog just like you did for water@2x.jpg.

Although the player is going to be another SKSpriteNode, like the background, we can’t create it inside the didMove(to:) method. You see, we need to be able to write code to manipulate the player later on in our game, after all the scene has been set up, so we need to create a different kind of variable for it.

Variables you create inside methods are available only inside those methods; as soon as they finish, the variables are no longer available to us. We want a variable that’s available everywhere in our game scene, so all we need to do is create it outside our methods. Swift calls these properties, but that’s just a special name for a variable created outside of methods.

Note: Although you create properties outside of methods, you can’t use them outside of methods. For example, we’re going to create the player node outside of methods, but we can’t set their position there too – that code needs to be inside a method.

Alright, enough chatter – please add this code directly before the override func didMove line:

let player = SKSpriteNode(imageNamed: "player-submarine.png")

That creates the player, but doesn’t position them on the screen or make them display. For that we need to two more lines, but remember: these need to go inside a method. So, please add these two to the didMove(to:) method, below the previous code:

player.position.x = -400
addChild(player)

If you run the code now you should be able to see the player’s submarine, but if you look carefully you might see they are behind the bubbles, which doesn’t look right.

This happens we haven’t altered the Z position of either the player or the particle emitter we created, so they both have the default Z position of 0. As a result, SpriteKit is free to decide which one gets drawn first, and it might change over time.

To fix this properly we’re going to give the player’s sprite node a position Z position. If you remember, we gave the background image a Z position of -1 so that it gets drawn behind other things. Well, now we’re going to give the player’s submarine a Z position of 1 so that it gets drawn in front of the bubbles.

Find this line of code:

player.position.x = -400

And add this directly after:

player.zPosition = 1

Much better!
