Starting with the background

The first step in this project is really easy: we’re going to draw a background image that takes up the full screen.

If you look inside GameScene.swift you’ll find some empty methods ready to be filled, including one called didMove(to:). This gets called when your game scene is presented on the screen, so it’s a good place to do any work required to make it work.

The background image will be created using a class called SKSpriteNode. SpriteKit game scenes are organized in a tree structure that make it easy and efficient to manage lots of things at the same time. Sprite nodes are SpriteKit nodes that hold image data, like our background image or the animals we’ll be adding soon.

When working with nodes, the four main things you need to worry about are:

  1. The node’s parent node. Remember, this is a tree, which means all nodes have a parent node, right the way up to your game scene, which controls everything.
  2. The node’s position. This is specified as a CGPoint containing X and Y coordinates. This position is relative to the node’s parent, and measured from the node’s center.
  3. The node’s Z position, which decides whether it is drawn above or below other nodes. Nodes with higher-numbered Z positions appear on top of nodes with lower-numbered Z positions.
  4. The node’s name, which lets us refer to it uniquely later on.

For this background image we don’t need anything special. In fact, we don’t even need to give it a position – I’ve made it the full size of our screen, and the default position of X:0 Y:0 will automatically place it in the center so it fills up the full screen.

First, press Alt+Cmd+1 to activate Xcode’s project navigator in the left of its window. This lists the contents of your project, and you should see Assets.xcassets in there – this is your asset catalog, which is where you can store all the images you want to use in your game. Please select it now, to open it for editing.

Second, find the project assets that came with this book. In there you should see background-leaves@2x.png, and I’d like you to drag that file from Finder into your Xcode asset catalog – just below where it says “AppIcon”.

The @2x part of the filename is there because it’s an image designed for retina iPads. iOS will automatically find and load the correct resolution of assets when we request them, so when we want to reference this image in code we’ll just say “background-leaves” and iOS will do the rest.

Let’s try that now – please add this code to the didMove(to:) method:

let background = SKSpriteNode(imageNamed: "background-leaves")
background.name = "background"
background.zPosition = -1
addChild(background)

That loads the background-leaves@2x.png image from our asset catalog, names it “background”, tells it to render at a Z position of -1, then adds it to the game scene. The default Z position is 0, so a Z position of -1 will put the background behind other things.

Try running the game now to make sure everything works, and all being well you should see the leaves image taking up all the space.
