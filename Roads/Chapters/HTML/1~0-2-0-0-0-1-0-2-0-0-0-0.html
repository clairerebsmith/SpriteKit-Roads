Wrap up

You’ve made it to the end of the first project – good job! Along the way you practiced some valuable skills:

  • How to use SKSpriteNode to draw graphics to the screen.
  • How to use Z position to draw one node above others.
  • How particle systems can be used to provide special effects like fire, smoke, and dust.
  • How to use Core Motion to let the user control the game by tilting their device.
  • How to use the Timer class to run code at regular intervals.
  • How to use Int.random(in:) to generate random numbers in a specific range.
  • How to create physics bodies out of textures.
  • How to use contact test and category bitmasks to control physics interactions.
  • How to use an SKLabelNode to show scores on the screen.
  • How to use an SKAudioNode to play looping music, and SKAction to play one-off sounds.
  • How to present new scenes after a delay.
  • And more!

In short, you’ve learned a lot, and you’ve got a finished game you can carry on developing further. Even better, it was a game you helped shape with each choice you made along the way – if you feel like it, you could even go back through this project and make different choices to see how your game could have turned out!

Challenges

Here are three ways you could take this game further if you fancy pushing your skills:

  1. Make multiple enemies, randomly chosen. Each time createEnemy() is called we always created the same enemy, but with a little input from Int.random(in:) it wouldn’t be hard to select one from several possible enemy graphics.
  2. Try to refactor createEnemy() and createBonus() into a single method to avoid code duplication.
  3. Make it so that tapping the screen causes the player’s motorbike to fire a weapon at the barrels. Perhaps make their number of shots limited, then using Timer to add one reload every few seconds.
