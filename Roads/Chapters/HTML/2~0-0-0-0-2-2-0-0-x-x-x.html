Adding sound effects

Now for my favorite part of any game project: adding sound and music. I like this part because you get results fast, and because it’s surprising how audio can really help a game feel more complete.

First, music: I picked out five tracks for you to choose from depending on how you want your game to feel. All five of them were created by Kevin MacLeod of https://incompetech.com/ and released under the Creative Commons: By Attribution 3.0 License (http://creativecommons.org/licenses/by/3.0/). In short, that means you’re free to use his music in your games as long as you give him credit somewhere.

The tracks are: Lobby Time, Easy Lemon, Cool Vibes, Truth in the Stones, and Night Cave. Try them all in the assets for this project, see which one you like, then drop it into Xcode’s project navigator. I chose Truth in the Stones, so when you see truth-in-the-stones.mp3 in my code please change it for whichever track you chose instead.

To make the music play we’re going to use SKAudioNode, which is SpriteKit’s awesome little audio player. It’s awesome because it automatically plays as soon as it’s added anywhere in our game scene, and because it automatically loops when it reaches the end – perfect for our music.

Like with the score node, we don’t want to add an SKAudioNode to the root of our game scene because that will cause problems with our grid of sprite nodes. So, instead we’ll add it to the background – put this code at the end of didMove(to:):

let music = SKAudioNode(fileNamed: "truth-in-the-stones")
background.addChild(music)

Remember to change “truth-in-the-stones” for whichever track you chose.

That’s music done already, so now we just need to add sounds when the user guesses rightly or wrongly. In the assets for this project I’ve included four correct sounds (correct-1.wav through correct-4.wav) and four wrong sounds (wrong-1.wav through wrong-4.wav) for you to choose from.

Pick any one correct and one wrong sound you like, and drag them both into Xcode’s project navigator. We aren’t going to use SKAudioNode to play these sounds, because SpriteKit has an even easier solution: an SKAction that plays one-off sounds.

We can run these actions directly on our game scene, so go ahead and add this line to the start of the correctAnswer() method:

run(SKAction.playSoundFileNamed("correct-1", waitForCompletion: false))

Make sure you change “correct-1” for whichever file you chose.

Now add this to the top of wrongAnswer():

run(SKAction.playSoundFileNamed("wrong-1", waitForCompletion: false))

Again, change “wrong-1” for whichever sound you chose.

Try the game now and I hope you’ll agree that sound and music makes a big difference!
