Wrap up

You’ve made it to the end of the second project – good job! Along the way you practiced some valuable skills:

  • How to use SKSpriteNode to draw graphics to the screen.
  • How to use Z position to draw one node above others.
  • How to use SKAction to animate changes to nodes.
  • How to use an SKLabelNode to show scores on the screen.
  • How to use an SKAudioNode to play looping music, and SKAction to play one-off sounds.
  • How to present new scenes after a delay.
  • And more!

In short, you’ve learned a lot, and you’ve got a finished game you can carry on developing further. Even better, it was a game you helped shape with each choice you made along the way – if you feel like it, you could even go back through this project and make different choices to see how your game could have turned out!

Challenges

Here are some ways you could take this game further if you fancy pushing your skills:

  1. Using your new-found knowledge of SKAction, can you make the grid animals fade in when they are created, rather than just appearing?
  2. We added a cap on itemToShow so the player can’t go beyond a certain level, but can you modify the code so they actually win rather than just continuing?
  3. The game timer always uses white text. Can you adjust it so that it becomes red when the player is running low on time?
