Wrap up

You’ve completed the second project – well done! Alongside having a finished game to play, you also practiced some useful skills:

  • How to use SKSpriteNode to draw graphics to the screen.
  • How to use Z position to draw one node above others.
  • How particle systems can be used to provide special effects like fire, smoke, and dust.
  • How to scroll background sprites at different speed to create the illusion of depth.
  • How to let the player control their submarine by tapping the screen.
  • How to use the Timer class to run code at regular intervals.
  • How to make Swift generate random numbers in a specific range.
  • How to create physics bodies out of textures.
  • How to use contact test and category bitmasks to control physics interactions.
  • How to use an SKLabelNode to show scores on the screen.
  • How to use an SKAudioNode to play looping music, and SKAction to play one-off sounds.
  • How to present new scenes after a delay.
  • And more!

One interesting technique you met this time was using invisible collision rectangles to track score. As you saw, this is just a neat way of re-using code you already know, and it means SpriteKit takes on most of the work

In short, you’ve come on a long way, and you’ve got a finished game you can carry on developing further. Even better, it was a game you helped shape with each choice you made along the way – if you feel like it, you could even go back through this project and make different choices to see how your game could have turned out!

Challenges

Here are three ways you could take this game further if you fancy pushing your skills:

  1. Show a game over sprite node when the player dies. I’ve provided gameOver@2x.png for just this purpose – add it to your asset catalog, then add it to the scene when the player dies. For bonus points try experimenting with SKAction to make it animate!
  2. Create a new particle system that’s permanently attached to the player’s submarine. With a little creative work in the particle editor you can add a convincing wave effect behind the submarine.
  3. If you’re looking for a challenging task, keep track of how many mine you’ve created, then for every 10 call invalidate() on the timer property and re-create it with a shorter time interval so that mine get created faster.
